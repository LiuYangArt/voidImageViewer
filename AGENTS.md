# voidImageViewer 协作指南

本文件定义本仓库内 AI 协作的项目级约束。除非用户明确提出其他目标，默认目标是用尽量小的改动维护一个轻量、稳定、原生的 Windows 图片查看器。

## 1. 项目定位

- 项目名称：`voidImageViewer`
- 语言与技术栈：原生 `C` + `Win32 API` + `GDI/GDI+` + 资源脚本
- 主要用途：尽快打开并显示常见图片格式，尤其关注 GIF / WEBP 动画播放准确性
- 当前主版本定义见 `src/version.h`

设计取向以简单、直接、可维护为优先，不要把这个项目往大型框架、抽象层、跨平台架构上推。

## 1.1 沟通约定

- 默认使用中文进行说明、计划、结论和评审。
- 需要用户手动验证时，写清楚操作步骤和预期结果。
- 需要先给方案时，优先给面向结果的说法，不只给抽象术语。

## 2. 开工前必读

在开始搜索、改代码或写方案前，优先按下面顺序建立上下文：

1. 先读 `.codebase_index.md`
2. 再看根目录结构和相关子目录
3. 按任务精准读取目标文件，不要先通读 `src/viv.c`

如果任务和图片加载、窗口行为、输入处理、播放逻辑有关，优先定位 `src/viv.c` 中对应函数再展开阅读。这个文件体积很大，必须按函数、关键字或调用链分段阅读。

## 3. 目录总览

- `src/`: 主程序源码。绝大多数业务逻辑都在这里。
- `res/`: `.rc` 资源、图标、对话框定义、版本资源。
- `vs2026/`: 当前优先使用的 Visual Studio 工程。
- `vs2019/`: 兼容旧版 VS 的工程。
- `vs2005/`: 更老的历史工程，除非明确需要兼容旧工具链，否则不要优先改这里。
- `libwebp/`: 内嵌的第三方 WebP 源码。
- `nsis/`: 安装器相关内容。
- `docs/plans/`: 规划文档。

## 4. 核心文件职责

- `src/viv.c`: 主程序入口、窗口消息、图片加载、播放、导航、命令、关联、UI 行为核心。
- `src/webp.c`: WebP 静态图/动画解码适配层，是“特殊格式专用解码模块”范例。
- `src/config.c` / `src/config.h`: 配置项定义、默认值、持久化入口。
- `src/ini.c`: ini 读写。
- `src/localization*.h` / `src/localization.c`: 本地化字符串与语言切换。
- `src/os.c`, `src/string.c`, `src/utf8.c`, `src/wchar.c`: 平台和基础工具函数。
- `res/voidImageViewer.rc`: 对话框、菜单、图标、版本资源。

## 5. 编辑原则

- 除非用户明确要求，否则不要主动引入 fallback 方案。
- 优先做局部改动，避免对 `src/viv.c` 做大范围“整理式重写”。
- 不要因为 `src/viv.c` 很大就强行拆模块，除非当前任务确实要求结构性重构。
- 新格式支持、特殊解码器、专用加载逻辑，优先参考 `src/webp.c` 的接入方式。
- 默认不要修改 `libwebp/` 第三方源码，除非问题已经明确定位到底层库，或者用户明确要求升级/修复第三方实现。
- 涉及选项、菜单、对话框、关联或用户可见文本时，必须同时检查：
  - `res/resource.h`
  - `res/voidImageViewer.rc`
  - `src/localization.h`
  - `src/localization_en_us.h`
  - `src/localization_zh_cn.h`
- 涉及配置项时，至少同时检查：
  - `src/config.h`
  - `src/config.c`
  - `src/ini.c`
  - 相关 UI 绑定代码

## 6. 构建与验证

- 优先使用 `vs2026/voidImageViewer.sln`
- 次选 `vs2019/voidImageViewer.sln`
- `vs2026/voidImageViewer.vcxproj` 直接编入 `libwebp` 的源码，不依赖单独预编译库
- 改动源文件、头文件、资源或新增文件时，注意同步更新 `vs2019` 和 `vs2026` 工程
- 如果改动安装器、关联或发布信息，再检查 `nsis/` 与版本资源

如果无法在当前环境实际编译，要明确说明“未完成编译验证”，不要假设工程一定通过。

## 7. 搜索与阅读策略

- 小任务先搜关键字，再读对应函数附近的几十到几百行。
- 大任务先看 `.codebase_index.md` 的“任务切入路径”。
- 针对大文件，只读取与当前任务相关的函数、结构体、枚举和调用点。
- 不要默认读取整个 `src/viv.c`。

推荐关键字：

- 图片加载：`IStream`, `Gdiplus`, `webp_load`, `frame_callback`
- 入口：`WinMain`, `main`, `_viv_main`
- 配置：`config_`
- 本地化：`LOCALIZATION_ID_`
- 资源控件：`IDC_`, `IDD_`
- 关联：`association`, `open with`, `shell`

## 8. 已知注意事项

- `src/viv.c` 约 400 KB，是明确的热点文件，也是最容易误改的地方。
- `vs2026/voidImageViewer.vcxproj` 中存在 `render*.h` 的陈旧头文件引用，但仓库里没有这些头文件，源码中也未实际包含。修改工程时不要误以为它们是当前缺失依赖。
- `libwebp/` 下有 `PaxHeaders.X/` 目录，属于第三方归档残留内容；除非用户明确要求清理第三方目录，否则不要顺手处理。

## 9. 提交前检查

在声称任务完成前，至少确认下面几点：

- 改动覆盖了所有必要的源码、资源、本地化或工程文件
- 新增文件已加入 `vs2019` / `vs2026` 工程（如果需要）
- 没有把无关的格式、功能或 UI 一起改进
- 文本、菜单和控件命名与现有风格一致
- 如果未编译或未手测，明确写出缺口
